# Contributors

This guide describes the development workflow that is verified in the Moongate repository. It covers the required toolchains, the solution layout, local validation, repository conventions, and the contribution path.

## Start here

1. [Set up the development environment](development-setup.md).
2. [Learn the repository layout](repository-layout.md).
3. [Build and test the change](build-and-test.md).
4. [Follow the repository conventions](conventions.md).
5. [Prepare a contribution](contributing.md).

For component relationships and runtime flow, use the [Architecture overview](../architecture/index.md). The [Server Guide](../server/index.md) covers installing and operating the server rather than developing it.
