# Moongate v2 - Current Status (2026-02-23)

## Summary
- Core server runtime remains stable: build and full test suite are green.
- Command system is now role-aware (`AccountType`) for both console and in-game dispatch.
- Interactive console now supports `Tab` / `Shift+Tab` autocomplete with extensible argument providers.
- `help` supports contextual usage (`help <command>`) in addition to full command listing.
- Documentation UX was refreshed: moon-first branding, custom Moongate palette, and `Fira Code` typography.

## Highlights Since 2026-02-19

### Command System Enhancements
- Added per-command authorization level:
  - `CommandDefinition.MinimumAccountType`
  - `RegisterCommand(..., minimumAccountType: ...)`
- Authorization policy at execution time:
  - Console source is evaluated as `AccountType.Administrator`
  - In-game source uses `GameSession.AccountType` (set at login)
- `LoginHandler` now maps account role into session runtime state.

### Command UX Improvements
- Added contextual help:
  - `help` -> full list
  - `help <command>` -> single command description
  - unknown help target -> explicit feedback message
- Added autocomplete framework:
  - `ICommandSystemService.GetAutocompleteSuggestions(...)`
  - optional per-command argument provider
  - `ConsoleCommandService` handles `Tab` / `Shift+Tab` candidate cycling

### Documentation / DocFX Updates
- Removed oversized logo from docs entry pages in favor of moon branding.
- Added custom DocFX theme override:
  - `docs/templates/moongate/public/main.css`
- Theme palette aligned with Moongate logo colors.
- Typography updated to `Fira Code` via Google Fonts.
- Public docs URL confirmed and documented:
  - https://moongate-community.github.io/moongatev2/
